System Administrator Tools

Below are some perMIT system web pages that will be useful to people who maintain qualifier hierarchies in the perMIT system or Funds/Fund Centers in SAP. See also miscellaneous authorization web pages.


Rules-based authorizations: Links to reports, UI for setting rules, etc.

Report on differences between the Approver authorizations in the PD Org and in the perMIT system:
(refreshed each day at 5 AM, 12 noon, and 4 PM)

  • Abridged report: Show differences for matchable Spending Groups only
  • Unabridged report: Show all differences (including spending groups that cannot be matched because the [spending group code] has not been included in the org. name field in the PD Org).

A clean-up aid for the SAP standard hierarchy of Funds/Fund Centers:
Each Fund Center in SAP's standard hierarchy of Fund Centers should have as its children either all Funds or all Fund Centers, not a combination of of the two. Authorized individuals can display a list of Fund Centers that break this rule.
Show Fund Centers that break the "one kind of children" rule

A clean-up aid for the Custom Fund Center hierarchy:
In most cases, a Fund Center should only be a member of one Spending Group. You can display a list of Fund Centers that break this rule. Certificates and a meta-authorization are required to use this web page.
Show Fund Centers that belong to more than one Spending Group

Show qualifiers with an inactive status. (These are usually qualifiers that should be deleted but must have their related authorizations deleted first.)

Run quarterly report statistics.

For authorized administrators only: Update perMIT system parameters for data feeds and other processes.

Maintain the list of Model 1 (Release Strategy) Funds Centers. (To use this feature, you need an authorization to 'MAINTAIN MOD 1 FC'.)

Make additions or changes to one of the Qualifier hierarchies. You are probably not authorized to use this Web page; this web page can only be used by a few Roles administrators.
Maintain qualifiers within perMIT system.

Roles users reports (for system administrators with 'VIEW ADMIN REPORTS' authorization):

  • Users whose directory information has changed (e.g., department or employment changes)
  • People who can create authorizations in the perMIT system.
  • New: Number of authorizations created by a person for a given time period.
  • Missing Roles usernames, i.e., users who can create authorizations, but are waiting for a perMIT system username/password.
  • Roles usernames, and the last time they connected to the database.
  • For specially-authorized administrators: Add or delete Roles usernames
